# /idea — 记录华哥的商业/产品想法

华哥会随时抛商业模式或产品想法，本命令**立即结构化记录**，不让他重述。

## 流程

1. **接收想法**
   - 华哥说"我有个想法..." / "我觉得..." / "以后可以做..."
   - 抓取关键词、核心点

2. **结构化整理**（按以下骨架）

   ```markdown
   # 想法：<一句话核心>

   ## 一句话核心
   <30 字内说清做什么、解决什么问题>

   ## 背景 / 灵感来源
   <华哥提到的场景、痛点、机会>

   ## 对比分析
   | 维度 | 我们 | 竞品 A | 竞品 B |
   |---|---|---|---|
   | 定位 | | | |
   | 核心优势 | | | |
   | 商业模型 | | | |
   | 风险 | | | |

   ## 风险点
   - 技术：xxx
   - 商业：xxx
   - 合规：xxx
   - 执行：xxx

   ## 待办（下一步）
   - [ ] 调研 xxx
   - [ ] 找华哥确认 xxx
   - [ ] 跑通 MVP 验证 xxx
   - [ ] ...

   ## 关联资产
   - RKR 文档：xxx
   - 已有项目：xxx
   - 类似想法：xxx
   ```

3. **保存**
   - 路径：`~/hermes/ideas/YYYY-MM-DD_<主题>_想法.md`
   - 主题：3-5 个关键词，用下划线连接

4. **汇报**
   - 发飞书对话回执
   - push 玉芬整理师归档到 RKR（打"想法"标签）

5. **不打断**
   - 华哥说完才动手
   - 过程不打断、不确认
   - 完整输出后让他看

## 参数

- `$ARGUMENTS`：可选，预先填入主题
