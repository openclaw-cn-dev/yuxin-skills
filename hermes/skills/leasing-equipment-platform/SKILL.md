# 租赁式设备运营平台设计方法论

## 触发条件

✅ **应该使用**：
- 用户提"租赁 / 按用量计费 / 立方水收费 / 处理量计费"
- "一体化移动箱 / 集成处理设备 / 黑盒化 / 只露进出水"
- "网络员 / 区域代理 / 兼职合伙人 / 分润"
- "数据飞轮 / 故障预测 / AI 自动控制 / 远程监控"
- "装配式光伏 / 投资人分润 / 2 年回本"
- 任何"硬件 + 服务 + 平台"型业务

❌ **不适用**：
- 纯软件 SaaS（无硬件实体）
- 一次性销售模式
- 纯代理/经销模式（无平台）

---

## 核心设计原则

### 1. 商业模式：装配式光伏模板（成熟参照）

```
【公司】= 平台方（重资产 + 全栈能力）
   ├─ 硬件自研（设备本体）
   ├─ 软件自研（AI 控制 + 计费分账 + 后台管理）
   ├─ 业务（销售/招商/客户管理）
   ├─ 技术（AI 优化/控制中心/运维）
   └─ 管理（计费/分润/合同/对账）

【投资人/代理】= 资金方（投钱分钱）
   ├─ 投入资金
   ├─ 不参与经营
   └─ 拿分润（后台可追踪）

【客户】= 用水/用量方（按立方/单位付费）
   ├─ 2 年回本定价
   ├─ 闲置期打折/转场调度
   └─ 二维档位定价（处理难度 × 出水标准）

【网络员】= 兼职运营（数据采集 + 现场巡视）
   ├─ 公司自聘 / 本地兼职 / 区域代理
   ├─ 按消费量分润（绑定长期利益）
   └─ 工作：测水质 + 输入数据 + 现场巡视
```

### 2. 计费公式（核心数学）

```
单价/单位 = 电费 + 设备折旧 + 利润

约束：
- 设备投入 2 年内必须回本（硬约束）
- 单价不能只看硬件成本 = 还要算机会成本 + 风险溢价
- 闲置期：打折收费（不能停收）或转场调度
```

### 3. 二维档位定价

```
        │ 出 水 标 准 →
原水    │ 尾水排放级  养殖级  饮用级
难度 ↓ │ ─────────────────────────────────
高      │   中/低      中      高
中      │    低        中      中/高
低      │   极低      低/中     中

投入设备不同 → 成本不同 → 立方单价不同
```

### 4. 设备设计：黑盒化（保密与安全）

**核心思想**：**只露进出水**——别人不能私自进入箱内

| 舱体 | 密封性 | 内容 | 维护方式 |
|---|---|---|---|
| **主舱** | 完全黑盒（GPS+锁+报警+日志）| 核心处理单元 | **远程调试**（控制中心 AI 自动）|
| **副舱/小窗** | 可开（带锁，方便开）| 填料补料 + 简单维护 | 现场/远程 |
| **拓展接口** | 标准化预留 | 光伏挂件/远程屏/备用电源 | 模块化叠加 |

**关键设计**：
- **维修维护必须远程化/分级权限**（华哥/工程师/客户方不同等级）
- **防拆机**：GPS + 锁 + 报警 + 日志（4 重）
- **主舱 + 副舱**模式：类似家用打印机墨盒仓——主墨盒要开，但内部机械保密

### 5. 三层运维体系

```
【L1】AI 自运维（7×24 实时）
   ├─ 全设备数据实时采集
   ├─ AI 自动控制（曝气/加药/循环）
   ├─ 故障预测系统（ML 模型）
   └─ 报警 → 控制中心调度

【L2】技术员定期巡检（物理维护）
   ├─ 周/月/季度
   ├─ 物理检查 + 清理 + 大保养
   └─ 故障预测响应

【L3】网络员日常（水质录入）
   ├─ 现场巡视
   ├─ 水体检测 → 数据输入处理箱
   └─ 上报异常
```

**关键洞察**：**网络员不是操作员**（不手动调参数）——**网络员是数据采集员**：
- **手动 = 输入数据**（水质检测 → 录入）
- **自动 = 处理控制**（控制中心 AI 算最优参数 → 下发）

**AI 控制闭环**：
```
网络员测水质 → 输入处理箱 → 上报控制中心
→ 控制中心 AI 算最优参数 → 下发到处理箱
→ 处理箱自动调曝气/加药/循环
```

### 6. 数据飞轮（核心增长引擎）

```
更多设备部署 → 越多运行数据 → 越优处理方案
        ↑                            ↓
客户口碑 ← 降本增效 ← AI 模型优化
```

**关键**：**所有设备运行数据全量采集**——水/电/药/温度/压力/水质/故障日志

### 7. 软硬全部自研

- **硬件自研**（EDAI 9 大设备类比）
- **软件自研**（AI 控制中心 + 计费分账 + 后台管理）
- **原因**：公司全栈负责才是"装配式光伏模式"的核心；外采意味着分润被切走

### 8. 标准化底座 + 模块化拓展

- **基础款**：只露进出水 + 主控面板
- **进阶款**：箱顶加装光伏板（绿电降本）
- **拓展接口**标准化（预留挂载点 + 电源接口）
- **未来可叠加**：远程屏 / 备用电源 / 4G/5G 回传 / 边缘计算盒

## 14 个核心问答模板（用于需求讨论）

当用户给模糊需求时，用这 14 个问题快速对齐：

### 业务核心（5 个）
1. **回本周期**？（影响设备档次 + 平台投入 + 计费底线）
2. **计费公式**？（电费 + 设备折旧 + 利润）
3. **档位维度**？（原水难度 × 出水标准 二维矩阵）
4. **黑盒化目的**？（保密 / 安全 / 维护 / 法律）
5. **运维角色**？（自聘 / 兼职 / 区域代理 + 分润激励）

### 模式（3 个）
6. **数据采集范围**？（全量 vs 抽样）
7. **运维分工**？（AI 自运维 / 技术员 / 网络员 三层）
8. **可拓展需求**？（光伏 / 远程屏 / 备用电源 / 边缘盒）

### 技术（3 个）
9. **核心软件**？（AI 控制 + 数据采集 + 故障预测 + 计费分账）
10. **自研/外采**？（默认全部自研，硬件也自研）
11. **控制中心能力**？（实时监控 + AI 优化 + 远程调度）

### 边界（3 个）
12. **客户类型**？（养殖 / 工业 / 市政 / 饮用）
13. **部署场景**？（室内 / 户外 / 移动 / 固定）
14. **法律合规**？（水质标准 + 排放许可 + 数据隐私）

## 关键陷阱（2026-06-17 实战教训）

1. **"手动较正功率"是误解**——华哥指"手动输入水质数据"，不是手动调参数
2. **网络员不是操作员**——是数据采集员 + 现场巡视员
3. **黑盒化 = 留独立窗口**——填料补料要方便，不能全密封
4. **"分档"不是单维**——是二维矩阵（原水难度 × 出水标准）
5. **网络员激励 = 消费分润**——不是工资，是绑定长期利益的消费刺激
6. **数据飞轮是核心增长引擎**——不是监控，是越用越聪明
7. **2 年回本是硬约束**——不能改，是商业模式的基础
